Introduction

Pocket M3N is a simple open source Max Margin Markov Networks (M3Ns) package, developed for practical sequence labeling tasks in NLP research.

Highlights

  • No order limitation: Any n-gram features can be adopted for training and testing.
  • Use Struct Sequential Minimal Optimization for training.
  • Can perform n-best outputs.
  • Can output marginal probabilities for all candidates.
  • Support null features

Usage

  • Testing Pocket M3N on your computer

After downloading Pocket M3N package, unzip it, and switch to the directory that contains this document.
If you are in Windows platform, type the following command to learn model:
m3n learn template train model
After learning, type the following command to test:
m3n test model key result
When you see "label precision:0.943303", this means Pocket M3N has run correctly on your computer.

If you use Linux or cgywin, you could follow the steps below:
Type command to generate m3n:
make
Then type command to test:
./m3n learn template train model
./m3n test model key result
Then you will see "label precision:0.943303", testing success!

  • Training

To use Pocket M3N for training model, use command like "m3n learn template train model". Here 3 file names should be given one by one: template file, train file, model file. In this example, the 3 names are "template","train","model". The first 2 files should be prepared, and the last file is generated by Pocket M3N.

  • Training file format

Training data should be arranged like tables in the file. Each table denotes a labeled sequence. Use enter character to separate the tables. Each row in the table denotes the observation(s) and the labels for the corresponding token, separated by tabular character.   • Template file format

Each line in the template file denotes one template. Each template is designed in the format:
w0%x[i1,j1]w1%x[i2,j2] ... w(m-1)%x[im,jm]wm%y[k1]%y[k2]...%y[0]
Bold parts are customized parameters. The first index (i1,...,im) of x specified the relative position from the current focusing token, while the second index (j1,...,jm) of x specified absolute position of the column. w0,...,wm are customized strings. Index of y specified the label of the token in the relative position. Here are some attentions:

    • Indexes of y in each template (k1,...,kn) should be arranged in ascending order, with kn = 0. Any templates with kn unequal to 0 can be regularized to the above format by subtract kn from the first index of each x (i1-kn,i2-kn,...,im-kn) and index of y (k1-kn,...,kn-kn)
    • Templates share the same y should be closed

Here is an example:
Training data

%x[-1,0]%y[0]                   if previous word is "remains", then current label is "B"
                         if previous word is "remains", then current label is "I"
                         if previous word is "remains", then current label is "O"
                         
%x[0,1]%y[0]                    if current pos is "RB", then current label is "B"
                         if current pos is "RB", then current label is "I"
                         if current pos is "RB", then current label is "O"
                         
%x[-1,0]fairly%x[1,1]%y[-1]%y[0]        if previous word is "remains", current word is "fairly", 
                         next word's pos is "JJ" then current label is "B", previous label is "B"
                         if previous word is "remains", current word is "fairly", 
                         next word's pos is "JJ" then current label is "B", previous label is "I"
                         ...

Here , first illegal template ended with y[1], it should be regularized. Second templates are illegal since the first and third share the same y, they should be arranged closedly.

  • Null features

In some case, features could be null. For example, in Chinese word part of speech tagging, if you choose the second Chinese character in word as feature, then for single character words, this feature is null. In such case, set this unit with "" (null string).

  • Options in m3n learn command

Several training options could be added to Pocket M3N, as a more complex case, type command:
m3n learn -i 100 template train model
Here "-i 100" tells Pocket M3N to train no more than 100 iterations. Defaultly, this parameter is 10.
All the training options are given below:

option

type

default

meaning

-h

Print help message.

-c

double

1

Slack variables penalty factor.

-f

int

0

Frequency threshold. Features occur less than the threshold are eliminated. Usually, set "-f 1" to use all features, when features are too many to store in the memory, set higher value to eliminated rare features.

-k

int

0

Kernel type. Set "-k 0" to select linear kernel, set "-k 1" to select polynomial kernel, set "-k 2" to select rbf kernel, set "-k 3" to select neural kernel

-s
-d
-r

double
int
double

1
1
0

If kernel type is linear kernel, all the 3 parameters are invalid, k(a,b)=&lta,b>. If kernel type is polynomial kernel, k(a,b) = (s*&lta,b>+r)^d. If kernel type is rbf kernel, k(a,b)=exp{-s*||a-b||^2}. If kernel type is neural kernel, k(a,b)=tanh(s*&lta,b>+r)

-i

int

10

Max iteration number.

-e

double

0.000001

Controls the training precision

-o

int

0

With higher value, more information is printed. When training data are too large, you could set this option with high value to see the detailed training process.

So you could try several other examples:
m3n learn -c 5 -i 50 template train model
m3n learn -k 1 -s 1 -d 2 -r 1 -i 50 -o 0 -e 0.0000001 -c 1 template train model

  • Continue training

Since the training time is always too long, it is difficult to train sufficiently in a single stage. Pocket M3N provides option "-x" for continued training based on a previous training model. For example:
m3n learn -i 50 template train model
is equvalent to the following 3 commands:
m3n learn -i 10 template train model
m3n learn -x -i 20 template train model
m3n learn -x -i 20 template train model
When you use this option, all the training options except "-i","-o" are loaded from model file.

  • Testing

To use Pocket M3N for testing, type command "m3n test model key result". Here 3 file names should be given one by one: model file, key file, result file. In this example, the 3 names are "model","key","result". The first 2 files should be prepared, and the last file is generated by Pocket M3N.

  • Key file format

Key file format are exactly the same as train file.

  • Result file format

For the simplest case "m3n test model key result", the result file adds one column to key file, which is the label predicts by Pocket M3N

  • Options in m3n test command

All the testing options are given below:

option

type

default

-m

int

0

0 or 1, if "-m 1", M3N will calculate the marginal probability for each label.
